Challenges

Notice that the first three links in the above header are devoid of "you are here" information. This is common with "copy and paste" HTML navigation. The SHTML "include" removes the necessity of copying and pasting between files, which is good, but would seem to guarantee that this problem would persist, which is not so good. "You are here" info is very helpful to visitors.

The "dumb" header

This is the header used in the first three files:

<!--#include file="header.txt" -->

It produces a simple, standardized set of links on each page, including an unnecessary link to the current page itself. This is the condition the "Smart Includes" header seeks to overcome...
